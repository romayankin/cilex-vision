## Key decisions

- The environment still has no GPU, Triton instance, or populated `data/eval/` detector manifest, so I did not try to fake a real bake-off run. The report is explicitly a published-benchmark proxy.
- I used the protocol formula anyway, but only after making the assumptions explicit:
  - `night_AP` is neutralized across all three candidates because no primary-source night benchmark was found
  - YOLOv8-L `AP_small` is neutrally imputed from the other published small-object slices because I could not find a primary-source COCO `AP_small` slice for YOLOv8-L
- Throughput on the target 24 GB GPU class is treated as an inference from:
  - `docs/triton-placement.md` for the YOLOv8-L deployment anchor
  - official YOLOv9 / RT-DETR published speed numbers for relative scaling
- The proxy ranking ended up:
  - `yolov9c`: `0.6138`
  - `yolov8l`: `0.6118`
  - `rtdetr-l`: `0.6104`
- Because the top lead is only `0.0020`, the protocol’s `0.0200` clear-winner rule is not met, so the recommendation falls back to the safe default `YOLOv8-L`.

## Patterns established

- Comparison tooling now supports two modes cleanly:
  - live MLflow mode for real bake-off runs
  - offline JSON mode for published-benchmark proxies
- Chart generation is now self-contained and dependency-light:
  - SVG output only
  - no `matplotlib` / `numpy` requirement for comparison reports
- The runner and comparator both stay CLI-friendly:
  - `--help` works in a thin environment
  - expected operational failures still return concise `SystemExit` messages

## Gotchas for the next agent

- `data/eval/detector/manifest.json` is still missing. `run_detector_bakeoff.py` still fails fast by design until the evaluation set exists.
- The final report is not a substitute for the real detector bake-off. Missing items that still require live execution later:
  - Triton-backed throughput / latency
  - the protocol’s real `night_AP`
  - the real operational slice at threshold `0.40`
  - per-class AP on the pilot’s 7-class taxonomy subset
- `docs/bakeoff-results/detector-published-benchmarks.json` is the reproducibility anchor for this proxy report. If someone updates the comparison later, they should either update that JSON or replace the report with real MLflow-backed outputs.
- `.claude-task-context.md` is stale for this repo state. I still read it because the user asked, but the actual P1-E01 prompt lives in `.agents/prompts/P1-E01.md`.
