# P4-V04: Embedding Search — Handoff

## What was built

3 new files + 2 modified:

1. **`services/query-api/faiss_reader.py`** — Read-only FAISS index loaded from MTMC checkpoint snapshots in MinIO. Thread-safe via `threading.Lock`. Periodic refresh (default 5 min). Provides `search(vector, k)` and `get_embedding_by_track(local_track_id)`. Lazy `faiss` import. Deserializes the same `CheckpointData` pickle format used by `services/mtmc-service/checkpoint.py`.

2. **`services/query-api/routers/similarity.py`** — Two endpoints:
   - `POST /search/similar` — search by base64 image crop (decoded → OSNet via Triton → 512-d embedding) or raw 512-d embedding vector. Returns top-K similar tracks with cosine similarity scores, metadata, and MinIO signed thumbnail URLs. Supports filtering by camera_id, object_class, time range.
   - `GET /search/similar/{local_track_id}` — find visually similar tracks to a given track. Looks up the track's embedding in the FAISS index, searches, and excludes the query track from results.
   - Both endpoints restricted to admin + operator + engineering (viewer forbidden).

3. **`services/query-api/tests/test_similarity.py`** — 12 tests covering: vector search, wrong-dimension validation, image search with mocked Triton, image search without Triton (503), must-provide-input validation, track-by-ID search, track-not-found (404), camera/class filtering, time range filtering, viewer forbidden (POST + GET), engineering allowed.

Modified:
- **`services/query-api/main.py`** — Added `similarity` router import and `app.include_router(similarity.router)`.
- **`services/query-api/requirements.txt`** — Added `faiss-cpu>=1.7.4`, `numpy>=1.24.0`, `Pillow>=10.0.0`.

## Key decisions

| Decision | Rationale |
|----------|-----------|
| Read-only FAISS from checkpoint pickle | Never writes to the live MTMC index. Uses the same MinIO checkpoint format for zero-coupling. |
| Periodic refresh (5 min default) | Matches the MTMC checkpoint interval. Stale-read is acceptable for similarity search. |
| Over-fetch 3× for post-filtering | FAISS can't filter by metadata, so we fetch 3×k and filter in Python. Caps at 300 max. |
| Triton for image → embedding | Reuses the OSNet Re-ID model already deployed on Triton. Lazy import of tritonclient. |
| FakeFAISSReader in tests | Tests run without faiss or Triton. Separate FakeTritonClient fixture for image search tests. |
| Self-match exclusion on track search | GET by track_id fetches k+1 and filters out the query track itself. |
| Viewer role forbidden | Similarity search exposes embedding-level data. Restricted to operator+ per RBAC matrix. |

## Patterns followed

- Same router pattern as detections.py, lpr.py: `APIRouter` with `require_role` dependency.
- Same `get_current_user` + audit tagging pattern on every endpoint.
- Same `generate_signed_url` for MinIO thumbnail URLs.
- Same Pydantic response models with Optional fields.
- Same FakePool/conftest.py fixture pattern extended with FakeFAISSReader.
- Lazy imports for faiss, tritonclient, PIL (tests run without them).

## Gotchas

1. **Triton client must be set on `app.state.triton_client`.** The similarity router expects `request.app.state.triton_client` for image search. This needs to be wired up in `main.py` lifespan (or via configuration) before image search works. Vector search and track-by-ID search work without Triton.

2. **FAISS reader must be set on `app.state.faiss_reader`.** The lifespan in `main.py` does not yet create the FAISSReader. Production deployment needs to instantiate it with MinIO credentials and attach to app.state during startup.

3. **Checkpoint format coupling.** FAISSReader deserializes the same pickle as `services/mtmc-service/checkpoint.py`. If `CheckpointData` changes fields, the reader may break. The reader uses `getattr(checkpoint, "track_map", {})` defensively for forward compat.

4. **Post-filtering can reduce result count below k.** If filters are restrictive and the index is small, fewer than k results may be returned. The over-fetch factor (3×) mitigates this for most cases.

5. **Image search adds latency.** The crop → Triton roundtrip adds ~10-50ms depending on network. Vector search is ~1ms for typical index sizes.
