# P3-V03: Customer Admin UI — Handoff

## What was built

Extended the Next.js frontend at `frontend/` with 6 admin sub-pages and 2 reusable components:

### Pages
1. **`app/admin/page.tsx`** (updated) — Admin landing page with links to all sub-pages and quick-access Grafana dashboard cards.
2. **`app/admin/cameras/page.tsx`** — Camera CRUD: list cameras from topology API, add via CameraForm, inline edit, delete with confirmation, status badges.
3. **`app/admin/topology/page.tsx`** — Visual topology editor: SVG graph via TopologyEditor component, edge creation modal with transit time input, camera detail panel, edge disable on click.
4. **`app/admin/retention/page.tsx`** — Retention policy table with editable days for each data class (raw video 30d, clips 90d, metadata 365d, thumbnails 30d, debug 30d, audit 730d). Local state only — no backend endpoint yet.
5. **`app/admin/users/page.tsx`** — Role definitions (admin, operator, viewer, engineering) and permission matrix table. Placeholder — actual user CRUD requires auth service.
6. **`app/admin/health/page.tsx`** — Embedded Grafana iframes (6 panels) in 3-column grid with refresh button. Uses `NEXT_PUBLIC_GRAFANA_URL` env var.
7. **`app/admin/calibration/page.tsx`** — Lists cameras with calibration status columns (pass-through, miss rate, false trigger). "Run Calibration" shows CLI instructions for `edge_filter_calibration.py`.

### Components
8. **`components/CameraForm.tsx`** — Controlled form for camera create/edit with fields: camera_id, name, zone_id, latitude, longitude, location_description. Validates required fields.
9. **`components/TopologyEditor.tsx`** — SVG-based visual editor with circular layout. Camera nodes show ID, zone, status color. Edges show transit time labels with solid/dashed for enabled/disabled. Click node to select, right-click to start edge link, click edge label to remove.

### Updated files
10. **`lib/api-client.ts`** — Added `apiMutate` helper for POST/PUT/DELETE, full topology types (`CameraNode`, `TransitionEdge`, `TopologyGraph`, `CameraCreateRequest`, `EdgeCreateRequest`), and write methods (`addCamera`, `removeCamera`, `upsertEdge`, `getTopologyGraph`).
11. **`app/layout.tsx`** — Admin nav link now highlights on any `/admin/*` path (was exact `/admin` only).

## Key decisions

| Decision | Rationale |
|----------|-----------|
| `apiMutate` as separate helper from `apiFetch` | Write operations need method, body, and Content-Type header; reads only need GET with query params. Keeps `apiFetch` unchanged for backward compatibility. |
| Circular layout for TopologyEditor | Works for any camera count without manual positioning. Real floor-plan SVG overlay is a future enhancement. |
| Edge disable via upsert (not delete) | No DELETE endpoint for edges in the topology API. Setting `enabled: false` preserves the edge for potential re-enable. |
| Retention page stores in local state only | No backend endpoint for retention config. The note explains that actual enforcement is via MinIO lifecycle and TimescaleDB retention policies in infrastructure. |
| Users page is informational only | Auth service not yet built. Shows role definitions and permission matrix from privacy-framework.md so admins can reference the RBAC model. |
| Calibration page shows CLI instructions | No REST API for triggering calibration. Links to the existing `edge_filter_calibration.py` script with pre-filled camera-id argument. |
| `DEFAULT_SITE = "site-01"` hardcoded | Single-site MVP. Multi-site would need a site selector; the topology API already accepts site_id as a path param. |

## Patterns followed

- `"use client"` at top of every page/component (Next.js App Router with hooks)
- Pages own data fetching (`useEffect` + `useCallback`), components are pure presentation
- `isAdmin(role)` check at top of every admin page, returns early with access-denied message
- Loading/error/content rendering pattern from P2-V05
- Tailwind CSS only for all styling
- `apiFetch` / `apiMutate` through `lib/api-client.ts` for all API calls

## Gotchas

- The `DEFAULT_SITE` is hardcoded to `"site-01"`. Multi-site deployments will need a site picker on the admin pages.
- TopologyEditor uses right-click (contextmenu) to start edge creation. This may not work well on touch devices.
- The health page iframes require Grafana to be configured with `allow_embedding = true` in `grafana.ini`. Without this, iframes will show a blank page or error.
- Edge "removal" actually disables the edge (sets `enabled: false`). There is no way to fully delete an edge through the current API.
- The iframe `eslint-disable` comment in health/page.tsx is necessary because the iframe gets its title from the Grafana embed.

## Open questions

1. **Multi-site support**: Should the admin pages get a site selector dropdown, or is single-site sufficient for the pilot?
2. **Calibration API**: Should a REST endpoint be added to trigger calibration from the UI, or is CLI-only acceptable?
3. **User CRUD**: When will the auth service be built to support actual user management?
4. **Retention API**: Should retention config be persisted to the database, or is infrastructure-level config sufficient?
5. **Floor plan**: The topology editor uses algorithmic layout. Should it support uploading a floor plan SVG for manual camera positioning?
