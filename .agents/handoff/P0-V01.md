# Handoff: P0-V01 — Throwaway 1-Camera Prototype

## What was built

- `scripts/prototype/demo.py` (382 lines) — Single-file throwaway prototype:
  - OpenCV `VideoCapture` for RTSP or webcam (env var `CAMERA_URL`, default `"0"`)
  - `ultralytics` YOLOv8n at ~5 FPS with 0.40 confidence threshold
  - COCO class IDs mapped to the 7 taxonomy classes (person, car, truck, bus, bicycle, motorcycle, animal); all other COCO classes ignored
  - Bounding box overlay with class-colored labels
  - SQLite storage for detections (timestamp, class, confidence, bbox)
  - Flask at `:5000` with three views:
    - `/stream` — MJPEG stream of annotated frames
    - `/api/detections` — JSON of last 50 detections
    - `/api/chart` — JSON of per-minute detection counts (last 30 min)
  - `index` page renders all three: live feed, detection table, bar chart (Chart.js, auto-refresh every 3s)
- `scripts/prototype/requirements.txt` — 4 dependencies (flask, opencv-python-headless, ultralytics, numpy)
- `scripts/prototype/Dockerfile` — Self-contained image: python:3.11-slim, OpenCV system deps, pre-downloads YOLOv8n weights at build time
- `scripts/prototype/README.md` — Clearly marked as disposable, includes quick start, Docker usage, env var table, taxonomy mapping table

## Key decisions and why

- **OpenCV not GStreamer**: The task explicitly requires OpenCV for simplicity. The real edge-agent (P1-V01) uses GStreamer with decodebin, appsink, and exponential backoff — none of that applies here.
- **YOLOv8n not YOLOv8-L**: The small model (~6MB) downloads fast, runs on CPU, and is sufficient for a demo. Production uses YOLOv8-L via Triton with TensorRT.
- **COCO-to-taxonomy mapping**: YOLOv8n outputs 80 COCO classes. Only 6 map directly to taxonomy classes (person, bicycle, car, motorcycle, bus, truck). The 10 COCO animal classes (bird through giraffe) all map to the single "animal" taxonomy class. Everything else is dropped.
- **SQLite with check_same_thread=False**: Threading is required because the detection loop runs in a background thread while Flask serves requests. SQLite is fine for a single-writer demo. The `_db_lock` serializes all access.
- **MJPEG not WebSocket**: Simpler, works in all browsers via `<img src="/stream">`, no JS needed for the video feed. Good enough for a throwaway.
- **Chart.js from CDN**: Avoids bundling or building frontend assets. The page polls `/api/chart` every 3 seconds and redraws.
- **opencv-python-headless**: No GUI deps needed — the demo is browser-based, not a desktop app. Smaller image.
- **Inline HTML template**: Single `render_template_string` call keeps everything in one file as required.

## Patterns established

- None intentionally. This is a throwaway. No patterns from this prototype should carry into production services.

## Gotchas

- **Not async**: The real pipeline is fully async (asyncio + asyncpg + NATS). This prototype uses threads + SQLite + Flask. Do not reference this code for production patterns.
- **No reconnection backoff**: The detection loop does a naive 1-second sleep and re-opens on frame read failure. The real edge-agent has exponential backoff (1s → 60s). Acceptable for a demo.
- **No auth, no HTTPS**: Flask runs in plain HTTP with no authentication. The prototype is for local/demo use only.
- **SQLite datetime('now') uses server time**: The `get_per_minute_counts` query uses `datetime('now')` which is UTC on most systems but depends on SQLite's compile-time config. Detection timestamps are always UTC via `datetime.now(timezone.utc).isoformat()`.
- **YOLOv8n downloads on first run**: If not using Docker (which pre-downloads), the first run will download ~6MB from ultralytics. Requires internet access.
- **No Prometheus metrics**: The real services expose `/metrics`. This prototype doesn't — it's throwaway.

## Open questions or known limitations

- **Webcam on Docker**: Passing `/dev/video0` into Docker requires `--device` flag and may not work on macOS/Windows. The README notes this for Linux only.
- **No frame timestamp from camera**: Unlike the real edge-agent which stamps `edge_receive_ts` immediately after `read_frame()`, this prototype uses wall-clock time at detection time. Acceptable for a demo.
- **No test file**: The task spec doesn't list tests as a deliverable and the prototype is explicitly throwaway, so no pytest suite was written.
