# P2-X02 — Operations Runbooks

## What I built

- `docs/runbooks/incident-response.md`
  - all current Prometheus alerts grouped by category
  - per-alert severity, plain-English meaning, dashboard URL, diagnosis commands, fix commands, and escalation path
- `docs/runbooks/scaling.md`
  - procedures for adding cameras, GPU nodes, Kafka capacity, TimescaleDB capacity, MinIO capacity, and stateless service instances
- `docs/runbooks/backup-restore.md`
  - backup coverage, commands, restore order, RPO/RTO targets, cron schedule, and restore-test requirements
- `docs/runbooks/camera-onboarding.md`
  - physical install, RTSP validation, edge config, topology registration, calibration, monitoring checks, and rollback
- `docs/runbooks/service-restart.md`
  - per-service restart order, restart commands, state recovery notes, health verification, and full-stack restart order

## Key decisions

- **Pilot and multi-node paths are both documented**
  - The repo now has both a pilot single-host deployment and a multi-node Ansible deployment path.
  - The runbooks call out when to use `pilot-*` container names versus inventory-driven multi-node commands.

- **Prometheus query API is used where host ports are not published**
  - Several pilot application services are scraped by Prometheus but do not publish their metrics or health ports directly to the host.
  - The runbooks therefore prefer `curl http://localhost:9090/api/v1/query ...` or container logs over fake `localhost:<port>` checks.

- **Topology API is preferred, but SQL fallback is documented**
  - The topology router exists and is mounted under the Query API, but operator auth bootstrap for a ready-made admin cookie is still environment-specific.
  - The onboarding runbook keeps the API path as the preferred method and adds direct SQL as the fallback so operators are not blocked.

## Patterns followed

- Same YAML frontmatter and structure as `docs/runbooks/model-rollout-sop.md`
- Copy-pasteable commands throughout
- Explicit Grafana dashboard URLs and exact alert names from `infra/prometheus/rules/`
- Plain operator language rather than developer shorthand

## Gotchas

- `P2-O02` handoff was referenced by the task prompt but does not exist in `.agents/handoff/`. I used the current repo state and the storage-related deliverables that are present instead.
- `attribute-service`, `event-engine`, and `clip-service` do not currently have a guaranteed host-published health endpoint in the generic Ansible deployment path. The restart runbook uses logs and Prometheus presence instead.
- `query-api` topology writes require an admin cookie. The runbook documents the exact API calls but keeps SQL fallback instructions because the repo does not ship a first-class operator login flow.
