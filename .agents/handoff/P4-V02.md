# P4-V02: Jetson Edge Agent — Handoff

## What was built

6 new files + 2 supporting files:

1. **`services/edge-agent/jetson/jetson_pipeline.py`** — Full Jetson edge pipeline: GStreamer RTSP decode → motion filter → TensorRT inference → NATS publish Detection protobufs. Reuses base edge agent modules (motion_detector, local_buffer, nats_publisher, rtsp_client). Publishes to `detections.edge.{site_id}.{camera_id}` — no MinIO frame upload. 8 Prometheus metrics with `jetson_` prefix.

2. **`services/edge-agent/jetson/jetson_detector.py`** — TensorRT inference wrapper for YOLOv8n. Letterbox preprocessing (numpy-only, no cv2), per-class greedy NMS, confidence filtering. Handles [1, 4+7, 8400] YOLOv8 output tensor. Thermal throttling warnings when latency exceeds threshold. Lazy imports for `tensorrt` and `pycuda`.

3. **`services/edge-agent/jetson/jetson_config.py`** — Pydantic BaseSettings with `JETSON_` env prefix. Extends base edge agent config pattern with DetectorConfig (engine_path, thresholds, thermal warning) and JetsonHardwareConfig (power_mode, dla_core). Same `from_yaml()` pattern.

4. **`scripts/jetson/export_jetson_model.py`** — CLI tool for PyTorch → ONNX → TensorRT engine export. Supports INT8 (with calibration dataset) and FP16. Uses IInt8EntropyCalibrator2 for quantization. Prints model stats after build.

5. **`infra/jetson/Dockerfile.jetson`** — Based on `nvcr.io/nvidia/l4t-pytorch:r36.3.0-pth2.1-py3`. Installs GStreamer, copies edge agent + jetson variant, generates protos (including detection.proto). Engine file mounted at runtime.

6. **`infra/jetson/flash-config.sh`** — Post-flash setup: MAXN power mode, max clocks, fan always-on, 8GB swap for 16GB variants, NVIDIA Docker runtime, directory setup.

Supporting:
- **`infra/jetson/gen_proto_jetson.sh`** — Proto generation including detection.proto (base gen_proto.sh only generates frame + common).
- **`services/edge-agent/jetson/__init__.py`** — Package init.

## Key decisions

| Decision | Rationale |
|----------|-----------|
| Separate `jetson/` subdirectory under edge-agent | Jetson variant imports base modules via sys.path; keeps flat layout within jetson/ |
| No MinIO dependency | Jetson sends detections directly to NATS — no frame upload. Massive bandwidth savings for edge deployment |
| numpy-only resize (no cv2) | Avoids OpenCV dependency on Jetson; nearest-neighbour resize is sufficient for YOLOv8 preprocessing |
| Per-class greedy NMS in numpy | Standard NMS without torchvision dependency; matches central inference behavior |
| Detection protobuf (not FrameRef) | Jetson publishes Detection messages matching the central detection.proto schema exactly |
| Proto ObjectClass enum values mapped explicitly | `_CLASS_NAME_TO_PROTO` dict maps class names to proto enum integers (1-indexed) |
| Shared detector instance across cameras | Single TensorRT engine/context shared by all camera pipelines — serialized via `asyncio.to_thread` |
| 2GB default buffer (vs 10GB base) | Jetson has less disk; detection messages are much smaller than frame blobs |
| `jetson_` Prometheus prefix | Distinguishes from base `edge_` metrics; adds inference_latency and per-class detection counters |

## Patterns followed

- Same flat service layout as base edge agent
- Same Pydantic BaseSettings with `from_yaml()` and env prefix
- Same GStreamer RTSP patterns (lazy `gi` imports, decodebin pipeline)
- Same NATS publisher reuse with per-camera subjects
- Same local buffer for NATS outages
- Same `asyncio.to_thread()` for blocking calls (TRT inference, RTSP read)
- Same structured JSON logging format
- Same proto serialization pattern (lazy imports inside function)
- Same `_pts_to_epoch` fallback for PTS-to-wall-clock conversion
- Same signal handling for graceful shutdown

## Gotchas

1. **Proto generation must include detection.proto.** The base `gen_proto.sh` only generates frame + common protos. The Jetson variant uses `gen_proto_jetson.sh` which adds detection.proto. If running locally, use the Jetson proto gen script.

2. **TensorRT engine is platform-specific.** An engine built on x86 won't work on ARM Jetson. The `export_jetson_model.py` script must be run on the target Jetson (or cross-compiled with matching TRT version). Engine files are not portable between JetPack versions.

3. **Detector is single-threaded.** `asyncio.to_thread` runs inference in a thread pool, but TensorRT's execution context is not thread-safe. The shared detector works because only one inference runs at a time. If parallel inference is needed, create one detector per camera.

4. **INT8 calibration needs representative data.** Without `--calibration-data`, INT8 export falls back to FP16 for uncalibrated layers. Collect ~500 representative images from the deployment site for best accuracy.

5. **No frame persistence.** Unlike the central pipeline, Jetson discards frames after inference. Debug/replay requires re-pulling from the camera. Consider adding optional frame persistence for debugging in production.

6. **Flash-config.sh requires root.** The script sets power mode, clocks, fan, and swap — all require sudo. Run once after JetPack flash, not on every boot (settings persist).
