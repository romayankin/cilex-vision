# Handoff: P0-D07 — Time Synchronization & Timestamp Policy

## What was built
- docs/time-sync-policy.md — full timestamp policy with Mermaid diagram
- infra/chrony/chrony-edge.conf — edge Chrony template
- infra/chrony/chrony-core.conf — core Chrony template
- services/monitoring/clock_drift_check.py — Prometheus drift collector
- infra/prometheus/rules/clock-drift.yml — alert rules

## Key decisions
- edge_receive_ts is THE authoritative timestamp for cross-camera work.
  MTMC MUST use edge_receive_ts, never source_capture_ts.
- source_capture_ts is advisory only — camera clocks are not trusted.
- Drift thresholds: WARN >500ms, CRITICAL >2000ms (pairwise between cameras).

## Patterns established
- Chrony templates use 3 NTP pool servers, makestep 1 3, rtcsync
- Monitoring scripts go in services/monitoring/ (not scripts/)
- Clock drift collector uses --targets JSON file pattern for camera-to-host mapping

## Gotchas for next agent
- The clock_drift_check.py script needs chronyc installed on the monitoring host
- The cmdallow in Chrony configs uses 10.42.0.0/16 as placeholder — replace
  with actual monitoring network CIDR before deployment
- If building the edge agent: ensure edge_receive_ts is stamped IMMEDIATELY
  when the frame arrives, not after processing. This is the trust anchor.
