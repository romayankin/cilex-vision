# P4-X01 — Commercial Documentation

## What I built

- `docs/commercial/product-overview.md`
  - executive summary and value proposition for non-technical buyers
  - covers detection, tracking, Re-ID, events, LPR, search, multi-site operations, deployment models, security, and integration posture
- `docs/commercial/feature-matrix.md`
  - commercial three-tier packaging guide for Basic, Pro, and Enterprise
  - positions feature entitlements by operational scope rather than by internal implementation detail
- `docs/commercial/datasheet.md`
  - procurement-oriented summary of supported inputs, deployment models, hardware guidance, network planning, storage planning, security posture, and interfaces
- `docs/commercial/use-cases.md`
  - four business-facing use cases:
    - retail analytics
    - campus security
    - traffic monitoring
    - warehouse management and safety
- `docs/commercial/faq.md`
  - stakeholder FAQ covering privacy, compliance, integration, scaling, security, and operations
- `docs/commercial/roi-calculator.md`
  - workshop-ready ROI template with inputs, formulas, outputs, and example scenario frames for 10-, 50-, and 100-camera opportunities

## Key decisions

- **Kept the messaging commercial, not architectural**
  - The task is for sales-facing stakeholders.
  - I avoided internal service topology, protocol internals, and implementation jargon except where procurement accuracy required it.

- **Used documented capabilities as the ceiling**
  - I based the docs on `docs/taxonomy.md`, deployment docs, privacy docs, cost-model docs, and relevant Phase 2/4 handoffs.
  - I did not introduce net-new claims such as facial recognition, named-person identity, or unsupported pricing commitments.

- **Made the feature matrix illustrative, not contractual**
  - The Basic / Pro / Enterprise table is a packaging guide.
  - Final packaging still needs commercial scoping per customer and deployment.

- **Avoided hard pricing**
  - The ROI template uses customer-side inputs and formulas instead of embedded license pricing.
  - The docs point readers to sales for commercial terms.

## Patterns followed

- concise Markdown with clean tables suitable for direct reuse in proposals or enablement material
- plain-language explanations oriented to procurement, security, operations, and executive readers
- capability statements anchored to current repo state and documented NFR targets

## Gotchas

- **Do not oversell the portal**
  - The commercial docs describe multi-site portal capabilities at a high level, but the current portal comparison page still uses derived/mock metrics and its site-management APIs are not yet implemented server-side.

- **Journey visualization is still partially backed**
  - The broader product story includes cross-camera journey workflows, but the repo still has an existing Query API gap around a dedicated MTMC journey endpoint.

- **No webhook promises**
  - The prompt mentioned integration points broadly, but I kept the docs API-first and avoided promising a generally available webhook feature because the current repo state does not establish that capability cleanly.
