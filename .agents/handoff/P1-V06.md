# Handoff: P1-V06 — Basic Query API

## What was built

- `services/query-api/main.py` — FastAPI application with CORS, lifespan-managed asyncpg pool and MinIO client, Prometheus metrics at `/metrics`, health/readiness endpoints. Runs via uvicorn.
- `services/query-api/config.py` — Pydantic Settings v2 with `QUERY_` env prefix; nested configs for Database (asyncpg DSN), MinIO (signed URL expiry), JWT (secret, algorithm, cookie name), CORS, Pagination.
- `services/query-api/metrics.py` — 6 Prometheus metrics: query_requests_total{method,endpoint,status}, query_latency_ms{method,endpoint}, query_db_latency_ms{query_type}, auth_failures_total{reason}, audit_writes_total, audit_errors_total.
- `services/query-api/schemas.py` — Pydantic response models: DetectionResponse (with BoundingBox), TrackSummaryResponse, TrackDetailResponse (with TrackAttributeResponse), EventResponse (with signed clip_url), UserClaims. All aligned with DB schema in services/db/models.py.
- `services/query-api/auth/jwt.py` — JWT verification from httpOnly cookie (`access_token`). RBAC with 4 roles: admin (all endpoints, all cameras), operator (detections/tracks/events, scoped cameras), viewer (detections/tracks/events, scoped cameras), engineering (detections/tracks only, scoped cameras). `get_camera_filter()` returns None for admin (no filtering) or list of camera_ids for scoped users.
- `services/query-api/auth/audit.py` — Starlette middleware logging every API request to `audit_logs` table via asyncpg INSERT. Captures user_id, action (HTTP method), resource_type, resource_id, query params, status code, latency, IP address. Skips health/metrics/docs endpoints. Audit write failures never block the response.
- `services/query-api/utils/db.py` — asyncpg pool creation and query helpers (`fetch_rows`, `fetch_val`) with db latency tracking.
- `services/query-api/utils/minio_urls.py` — MinIO signed URL generation. Parses `s3://bucket/path` URIs, returns presigned GET URLs with configurable expiry (default 1 hour).
- `services/query-api/routers/detections.py` — `GET /detections` with filters: camera_id, start/end time, class, min_confidence. Raw SQL with parameterized queries. TimescaleDB chunk exclusion via explicit `WHERE time >= $start AND time < $end`. Camera scope filtering via `WHERE camera_id = ANY($cameras)`. Pagination with offset/limit (max 1000).
- `services/query-api/routers/tracks.py` — `GET /tracks` with filters: camera_id, start/end, class, state. `GET /tracks/{local_track_id}` with joined track_attributes and camera scope check.
- `services/query-api/routers/events.py` — `GET /events` with filters: site_id (via cameras JOIN), camera_id, start/end, event_type, state. Converts `clip_uri` to signed MinIO URL in response.
- `services/query-api/Dockerfile` — python:3.11-slim, no special system packages needed.
- `services/query-api/requirements.txt` — 9 dependencies (fastapi, uvicorn, asyncpg, pydantic, pydantic-settings, pyyaml, prometheus-client, PyJWT, minio).
- `services/query-api/README.md` — endpoints, auth, config.
- `services/query-api/tests/conftest.py` — FakePool (asyncpg mock with sequential query result support), JWT test helper, httpx AsyncClient fixture.
- `services/query-api/tests/test_auth.py` — 10 tests: missing cookie 401, invalid token 401, expired token 401, valid token 200, wrong role 403, invalid role 401, role-based access for all 4 roles.
- `services/query-api/tests/test_detections.py` — 8 tests: empty result, multiple detections, field mapping (bbox x/y/w/h), pagination params, limit validation 422, min_confidence filter, class filter, null track_id.
- `services/query-api/tests/test_tracks.py` — 9 tests: empty result, returns tracks, field mapping, state/class filters, track detail with attributes, 404 not found, camera scope blocks access (scoped user), admin bypasses scope.
- `services/query-api/tests/test_events.py` — 9 tests: empty result, returns events, field mapping (metadata, timestamps), clip_url None when no MinIO, event_type filter, null clip_uri, null track_id, engineering 403, three timestamps present.

## Key decisions and why

- **asyncpg raw SQL, NOT SQLAlchemy ORM**: per task instructions, the read path uses raw parameterized SQL for TimescaleDB chunk exclusion performance. SQLAlchemy's ORM query builder would obscure the `WHERE time >= ... AND time < ...` predicate that enables chunk skipping.
- **JWT in httpOnly cookie, not Authorization header**: per security-design.md. The cookie name is configurable (default `access_token`). This prevents XSS-based token theft.
- **Camera scope in JWT claims**: the JWT payload includes `camera_scope: list[str]` which is used for `WHERE camera_id = ANY($cameras)` filtering. Admin role bypasses this entirely (sees all cameras). This avoids a DB lookup on every request.
- **Audit logging as middleware, not per-endpoint**: ensures every request is logged without developers having to remember to add audit calls. Failures are swallowed (logged but don't block the response) because audit writes should never degrade API availability.
- **`clip_uri` → signed URL conversion in events router**: the DB stores `s3://` URIs. The API converts these to signed MinIO GET URLs (1hr expiry) before returning to the client. Browsers never see raw MinIO credentials.
- **Separate count + data queries**: each list endpoint runs two queries: a `COUNT(*)` for total and a `SELECT ... LIMIT/OFFSET` for data. This supports proper pagination metadata. For TimescaleDB hypertables, both queries benefit from chunk exclusion via the time range predicate.
- **4 RBAC roles from privacy-framework.md draft**: admin, operator, viewer, engineering. The permission matrix is defined in `auth/jwt.py` as `ROLE_PERMISSIONS`. Engineering role can only access detections and tracks (not events) — intended for debugging.

## Patterns established

- **FakePool test pattern**: `FakePool` with `set_rows_sequence()` for testing endpoints that make multiple sequential DB queries (e.g., track detail: fetch track, then fetch attributes). This avoids heavy mocking.
- **Query builder pattern**: parameterized SQL with `param_idx` counter and `conditions` list. Filters are appended conditionally, then joined with `AND`. This prevents SQL injection and keeps the code readable.
- **Camera scope as a cross-cutting concern**: `get_camera_filter(user)` returns `None` (admin, no filter) or `list[str]` (scoped). Routers call this and append `camera_id = ANY(...)` when non-None. The track detail endpoint additionally checks the result row's camera_id post-fetch for single-item lookups.
- **Signed URL generation as a utility**: `utils/minio_urls.py` is reusable across any endpoint that needs to expose `s3://` URIs to clients.

## Gotchas

- **openapi.json stub**: the existing `services/query-api/openapi.json` stub was NOT replaced because FastAPI auto-generates the spec at `/openapi.json` at runtime. The static file would need to be generated by running the app and hitting the endpoint. It can be captured via `curl localhost:8000/openapi.json > openapi.json`.
- **Thumbnail URL is always None**: the track detail endpoint's `thumbnail_url` field is always None because there is no frame-reference table linking tracks to stored frame URIs. When such a table exists, the query can be added to look up the best-confidence detection frame URI and generate a signed URL.
- **No user/login endpoints**: this API only handles JWT verification, not issuance. A separate auth service (or admin CLI) would need to create JWTs with the correct claims (user_id, role, camera_scope). Password verification with bcrypt is not in scope here.
- **Audit log INSERT is not COPY**: the audit middleware uses a single `INSERT` per request, not COPY. This is appropriate because audit writes are low-volume (one per API request, not per detection) and need to be non-blocking.
- **Site filter on events requires cameras JOIN**: the `events` table has `camera_id` but not `site_id`. When `site_id` is passed as a filter, the router JOINs to `cameras` to resolve the relationship. This is safe because `cameras` is a small relational table.
- **No WebSocket streaming**: the kafka-contract.md mentions `query-api-events` consumer group for real-time event streaming. This is not implemented — the current API is pull-based only.

## Open questions or known limitations

- **No rate limiting**: the API has no request rate limiting. For production, a middleware or reverse proxy rate limiter should be added.
- **No cursor-based pagination**: current pagination uses offset/limit. For large result sets (millions of detections), cursor-based pagination using `WHERE time < $last_seen_time` would be more efficient and avoid the `COUNT(*)` overhead.
- **No caching**: responses are not cached. For dashboards polling the same queries, a Redis cache or HTTP cache headers could reduce DB load.
- **Privacy-framework.md is still a STUB**: the RBAC role definitions come from the draft section. When the full privacy framework is completed, the role permissions may need updating.
- **Query p95 latency NFR (< 500 ms)**: the `query_latency_ms` histogram is exported but no alerting is configured. The target from taxonomy.md §4 should be monitored via Prometheus/Grafana.
