# P4-O01: Multi-Site Infrastructure

You are working in /repo/infra/. Create multi-site Ansible/Terraform templates.

## Context

Commercial deployment requires deploying the platform across multiple customer sites with centralized management.

## Reading Instructions

Before writing any code, read:
- .agents/handoff/P3-O01.md — single-site deployment automation (once available)
- .agents/handoff/P1-O01.md — pilot infrastructure
- docs/security-design.md — per-site certificates, mTLS
- CONVENTIONS.md

## Implementation

1. Extend Terraform for multi-site:
   - terraform/modules/site/ — per-site infrastructure module (edge gateways, local storage)
   - terraform/modules/central/ — central infrastructure (Kafka cluster, DB, GPU pool)
   - terraform/environments/multi-site/ — multi-site orchestration
2. Extend Ansible for multi-site:
   - playbooks/add-site.yml — add a new site to existing deployment
   - playbooks/remove-site.yml — decommission a site
   - roles/site-edge/ — per-site edge infrastructure
   - inventory/multi-site.yml — multi-site inventory with per-site variables
3. Create scripts/deploy/site-onboarding.sh — end-to-end site onboarding automation

## Constraints

- Adding a site must not disrupt existing sites
- Per-site PKI isolation (per-site certificates)
- Must be idempotent
- Support heterogeneous hardware across sites
