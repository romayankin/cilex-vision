# P4-V04: Embedding Search

You are working in /repo/services/query-api/. Add FAISS-based visual similarity search API.

## Context

Enable operators to search by visual similarity: upload a crop or select a detection, find visually similar objects across all cameras.

## Reading Instructions

Before writing any code, read:
- .agents/handoff/P2-V02.md — MTMC service FAISS index, embedding format
- .agents/handoff/P1-V06.md — query API structure
- proto/embedding.proto — Embedding protobuf
- CONVENTIONS.md

## Implementation

1. Add to query API:
   - POST /search/similar — accept image crop (base64) or embedding vector, return top-K similar detections with thumbnails
   - GET /search/similar/{local_track_id} — find tracks visually similar to given track
2. Create services/query-api/routers/similarity.py:
   - Query FAISS index (read-only replica or checkpoint) for nearest neighbors
   - Return results with confidence, camera_id, timestamp, thumbnail signed URL
3. If image crop provided:
   - Run through Re-ID model (Triton gRPC) to get embedding
   - Search FAISS index with resulting embedding
4. Support filtering by time range, camera, class

## Constraints

- Read-only FAISS access — do NOT write to the live MTMC index
- Use a FAISS index snapshot from MinIO checkpoint
- Must pass `ruff check` and `mypy --strict`
