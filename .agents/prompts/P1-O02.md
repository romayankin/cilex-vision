# P1-O02: Observability Dashboards & Alerts

You are working in /repo/infra/grafana/ and /repo/infra/prometheus/.

Create Grafana dashboard JSONs (provisioned via grafana/provisioning/dashboards/):
1. dashboards/stream-health.json — per-camera uptime %, decode errors/min, frame gaps, motion duty cycle.
2. dashboards/inference-performance.json — FPS, inference latency p50/p95/p99, GPU util %, VRAM usage, Triton queue depth.
3. dashboards/bus-health.json — Kafka consumer lag per topic/group, NATS connections, bridge spool fill %, throughput.
4. dashboards/storage.json — TimescaleDB write rate, MinIO bucket sizes by tier, retention compliance.
5. dashboards/model-quality.json — confidence distribution per class, detection count per camera/hour, tracker ID-switch rate.

Create Prometheus alert rules in infra/prometheus/rules/:
- stream-alerts.yml: dropped_frames > 100/min, camera_offline > 5min
- inference-alerts.yml: latency_p99 > 200ms, vram_headroom < 15%
- bus-alerts.yml: kafka_consumer_lag > 10000, bridge_spool_fill > 80%
- storage-alerts.yml: timescaledb_write_errors > 0, minio_growth_anomaly
- clock-alerts.yml: clock_skew > 500ms (from P0-D07)
