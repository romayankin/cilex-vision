# P3-O01: Deployment Automation

You are working in /repo/infra/. Create Ansible/Terraform for multi-node deployment.

## Context

Moving beyond the single-node pilot to a multi-node production deployment requires proper automation.

## Reading Instructions

Before writing any code, read:
- .agents/handoff/P1-O01.md — existing Ansible playbooks for pilot
- infra/ansible/ — current playbook structure
- docs/security-design.md — mTLS, PKI requirements
- CONVENTIONS.md

## Implementation

1. Create infra/terraform/ — Terraform modules for:
   - terraform/modules/compute/ — GPU and CPU node provisioning
   - terraform/modules/network/ — VPC, subnets, security groups, camera VLANs
   - terraform/modules/storage/ — NVMe provisioning for spooling and FAISS
   - terraform/environments/production/ — production environment config
   - terraform/environments/staging/ — staging environment config
2. Extend infra/ansible/ for multi-node:
   - playbooks/deploy-multi-node.yml — orchestrate full multi-node deployment
   - roles/gpu-node/ — NVIDIA driver, container runtime, Triton
   - roles/edge-gateway/ — edge agent, NATS, local buffer
   - inventory/production.yml — multi-node inventory template
3. Create scripts/deploy/health-check-all.sh — verify all nodes and services healthy after deployment

## Constraints

- Terraform must be idempotent and support plan/apply workflow
- Ansible must be idempotent
- Support both bare-metal and cloud deployment
- Must not break existing pilot deployment scripts
