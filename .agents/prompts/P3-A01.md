# P3-A01: Continuous Annotation Pipeline

You are working in /repo/scripts/annotation/. Auto-export hard examples to CVAT for human review.

## Context

To continuously improve model quality, hard examples (low confidence, disagreements) should be automatically queued for human annotation.

## Reading Instructions

Before writing any code, read:
- .agents/handoff/P1-A01.md — CVAT setup, annotation projects
- .agents/handoff/P1-V07.md — debug traces (low-confidence detections)
- .agents/handoff/P3-E03.md — drift detection (once available)
- CONVENTIONS.md

## Implementation

1. Create scripts/annotation/hard_example_miner.py:
   - Query debug traces for low-confidence detections (< configurable threshold)
   - Query for high-disagreement frames (if shadow deploy active)
   - Sample N examples per day per class (configurable, default 50)
   - Download frame crops from MinIO
2. Create scripts/annotation/auto_create_cvat_tasks.py:
   - Create CVAT tasks from mined examples
   - Pre-fill with model predictions as annotation suggestions
   - Assign to annotator queue
3. Create scripts/annotation/annotation_feedback_loop.py:
   - Export completed annotations
   - Compare with model predictions
   - Compute model error rate on hard examples
   - Feed into retraining dataset (P3-V01)

## Constraints

- Must not overwhelm annotators: cap at configurable daily limit
- Do NOT modify CVAT or any service
- Must pass `ruff check`
