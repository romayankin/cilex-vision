# P3-V03: Customer Admin UI

You are working in /repo/frontend/. Add admin pages to the Next.js app.

## Context

This task extends the Search UI (P2-V05) with administration capabilities for camera onboarding, topology editing, and system health monitoring.

## Implementation

Add admin pages to the Next.js app:
1. /admin/cameras — CRUD cameras: add RTSP URL, assign to site, test connectivity, view health score
2. /admin/topology — visual topology editor: drag cameras on a floor plan SVG, draw edges between them, set zone boundaries, configure transit-time priors
3. /admin/retention — configure retention policies per data class per site
4. /admin/users — user/role management
5. /admin/health — embedded Grafana iframes for key dashboards
6. /admin/calibration — trigger edge filter calibration per camera, view latest scorecards

All operations via REST API to query-api admin endpoints. RBAC: admin role required for all admin pages.

## Constraints

- Do NOT modify services/query-api/ — only consume existing endpoints
- Reuse existing frontend component patterns from P2-V05
- Admin pages require admin JWT role
- Must pass `next lint`
