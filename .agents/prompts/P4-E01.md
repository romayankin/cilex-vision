You are working in /repo/scripts/load-test/. System-level performance validation at target scale.

## Context

Before declaring the platform production-ready, it must be validated at the target scale of 50-100 cameras.

## Implementation

1. replay_streams.py — use FFmpeg to replay recorded RTSP streams at real-time rate, multiplexed to simulate 50-100 cameras from a smaller set of recordings
2. measure_e2e.py — instrument every pipeline stage: ingest latency, decode latency, inference latency, tracker latency, DB write latency, query API response time. Collect p50/p95/p99 for each
3. chaos_scenarios.py — kill one Kafka broker mid-test, pause a consumer group for 30s, simulate edge WAN outage and replay
4. generate_report.py — produce comprehensive Markdown report: throughput achieved, latency percentiles per stage, resource utilization (CPU, GPU, RAM, disk, network) per component, bottleneck identification, cost model comparison (predicted vs actual). Include pass/fail criteria from NFRs in taxonomy.md
