# P2-O01: MTMC Infrastructure

You are working in /repo/infra/. Set up FAISS and GPU placement for the Re-ID models used by the MTMC service.

## Context

The MTMC service (P2-V02) requires FAISS for embedding similarity search and GPU resources for the Re-ID model. This task provisions that infrastructure.

## Reading Instructions

Before writing any code, read:
- .agents/handoff/P2-V02.md — MTMC service resource requirements
- .agents/handoff/P1-O01.md — existing Ansible playbook structure
- docs/triton-placement.md — GPU class definitions, VRAM budget
- infra/ansible/ — existing playbook patterns
- CONVENTIONS.md

## Implementation

1. Add FAISS installation to the MTMC service deployment:
   - faiss-cpu for pilot (no GPU requirement for index search at 4 cameras)
   - faiss-gpu provisioning playbook for scale deployment
2. Update Triton model repo with Re-ID model (OSNet-x1.0):
   - infra/triton/models/osnet_reid/config.pbtxt
   - Class B GPU placement per docs/triton-placement.md
3. Add MTMC-specific Prometheus metrics and Grafana dashboard:
   - infra/grafana/dashboards/mtmc-health.json — match rate, reject rate, FAISS index size, checkpoint lag, rebalance events
4. Add Prometheus alert rules:
   - infra/prometheus/rules/mtmc-alerts.yml — match_rate < threshold, checkpoint_lag > 10min, faiss_index_size anomaly
