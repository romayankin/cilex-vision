# P4-V02: Jetson Edge Agent

You are working in /repo/services/edge-agent/. Create a variant of the edge agent that runs inference on NVIDIA Jetson.

## Context

For deployments where central GPU capacity is limited, running detection on Jetson edge devices reduces bandwidth and central compute requirements.

## Reading Instructions

Before writing any code, read:
- .agents/handoff/P1-V01.md — current edge agent architecture
- docs/triton-placement.md — model inventory, TensorRT configs
- docs/taxonomy.md — confidence thresholds
- CONVENTIONS.md

## Implementation

1. Create services/edge-agent/jetson/ — Jetson-specific variant:
   - jetson_pipeline.py — GStreamer decode → TensorRT inference → motion + detection filter → NATS publish
   - jetson_detector.py — TensorRT runtime for YOLOv8n on Jetson (INT8 quantized)
   - jetson_config.py — Pydantic settings for Jetson constraints (memory, thermal)
2. Create scripts/jetson/export_jetson_model.py — export model to Jetson-compatible TensorRT engine
3. Create infra/jetson/Dockerfile.jetson — based on NVIDIA L4T base image
4. Create infra/jetson/flash-config.sh — JetPack configuration script
5. Only send detections + metadata to central (not frames) — massive bandwidth savings

## Constraints

- Must work on Jetson Orin NX 16GB and Jetson AGX Orin
- Reuse existing edge agent code where possible
- Must maintain same NATS publish format as central-inference mode
- Must pass `ruff check`
