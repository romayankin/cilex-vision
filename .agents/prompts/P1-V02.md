# P1-V02: Ingress Bridge Service

You are working in /repo/services/ingress-bridge/. Implement per ADR-001 (docs/adr/).

Python service:
- Consume from NATS JetStream durably
- Validate Protobuf schema via Confluent Schema Registry
- If payload >100KB, upload to MinIO, replace with URI reference
- Produce to Kafka with idempotent keys: {site_id}:{camera_id}:{capture_ts}:{frame_seq}
- When Kafka unavailable: spool to local NVMe (max 50GB)
- On recovery: drain spool with rate limit (1000 msg/s per site)
- Per-site rate limiter (default 500 msg/s)

Metrics: nats_consumer_lag, kafka_produce_latency_ms, spool_fill_pct, schema_rejection_total
Tests: including Kafka-kill chaos scenario (mock producer error → spool → recover → drain)
