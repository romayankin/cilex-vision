# P0-D09: Model Rollout & Cutover SOP

You are working in /repo. Create docs/runbooks/model-rollout-sop.md:

Four-stage model update process:
1. OFFLINE QUALIFICATION — eval on all slices, compare vs baseline, no regression >2%, log to MLflow.
2. SHADOW DEPLOYMENT — new model in shadow Triton, dual inference, comparison dashboard, 24h minimum.
3. SITE-LEVEL CUTOVER — all workers simultaneously, tracker state reset, embedding_model_version boundary enforcement, FAISS flush if Re-ID model changed.
4. POST-CUTOVER WATCH — 4h elevated monitoring, rollback trigger if >15% degradation.

Include per-stage: gate criteria, commands, rollback procedure, checklist template.
