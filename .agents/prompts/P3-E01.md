# P3-E01: Retraining Validation

You are working in /repo/scripts/evaluation/. Automated comparison of retrained vs production model.

## Context

After the retraining pipeline (P3-V01) produces a new model, this task validates it against the production baseline before shadow deployment.

## Reading Instructions

Before writing any code, read:
- .agents/handoff/P3-V01.md — retraining pipeline outputs, MLflow registry
- docs/bake-off-protocol.md — evaluation metrics
- docs/runbooks/model-rollout-sop.md — Stage 1 offline qualification
- CONVENTIONS.md

## Implementation

1. Create scripts/evaluation/validate_retrained_model.py:
   - Load retrained model and production baseline from MLflow
   - Run both on the full evaluation dataset (all slices: day/night/rain/crowded/occluded)
   - Compute per-class mAP, per-condition accuracy, latency
   - Generate comparison report: per-slice deltas, regression flags (>2% regression on any slice)
   - Output GO/NO-GO recommendation
2. Create scripts/evaluation/regression_checker.py:
   - Reusable function: compare two MLflow runs, flag regressions
   - Configurable threshold per metric
3. Integrate with retraining Makefile: `make validate` target

## Constraints

- Must use the same evaluation dataset splits as bake-offs
- Log all results to MLflow
- Must pass `ruff check`

## Expected Deliverables

- scripts/evaluation/validate_retrained_model.py
- scripts/evaluation/regression_checker.py
