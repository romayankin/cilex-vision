# P4-V03: LPR Module

You are working in /repo/services/lpr-service/. Create a license plate recognition pipeline.

## Context

License plate recognition adds vehicle identification capability, complementing the existing color attribute and Re-ID matching.

## Implementation

1. Create a service that consumes vehicle detections from tracklets.local:
   - Filter for vehicle classes only (car, truck, bus)
   - Quality gate: minimum plate region size (configurable), sharpness check
2. Two-stage pipeline:
   - Stage 1: plate detection — locate plate region within vehicle bbox
   - Stage 2: OCR — recognize characters from plate crop
3. Both stages via Triton gRPC
4. Store results: plate_text, confidence, country_format, bbox within vehicle
5. Expose via query API: search by plate text (exact, prefix, wildcard)
6. Alembic migration for lpr_results table

## Constraints

- LPR is an OPTIONAL module — must not break pipeline if disabled
- Legal compliance: document which jurisdictions allow ANPR
- Use Protobuf for inter-service messages
- Must pass `ruff check` and `mypy --strict`
