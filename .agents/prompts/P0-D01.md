## Requirements

Define:
1. **OBJECT CLASSES** — person, car, truck, bus, bicycle, motorcycle, animal.
   For each: one-line definition, minimum confidence threshold (suggest 0.4 for pilot).

2. **ATTRIBUTES** — vehicle_color (enum: red, blue, white, black, silver, green, yellow, brown, orange, unknown), person_upper_color, person_lower_color (same enum).
   For each: which object class it applies to.

3. **EVENTS** — entered_scene, exited_scene, stopped, loitering, motion_started, motion_ended.
   For each: trigger condition, close condition, required fields (start_time, end_time, duration, track_id, camera_id, clip_uri).

4. **NON-FUNCTIONAL REQUIREMENTS** — table: NFR | Target | Measurement.
   Include: e2e latency <2s, inference FPS 5-10, pilot cameras 4, retention (raw 30d, events 90d, metadata 1y), query p95 <500ms.

5. **EVENT STATE MACHINE** — Mermaid state diagram: new → active → stopped → exited → closed.

Format as structured Markdown with YAML front-matter (version, date).
