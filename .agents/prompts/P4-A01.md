# P4-A01: Multi-Site Dataset

You are working in /repo/scripts/annotation/. Aggregate evaluation datasets across multiple sites.

## Context

With multiple sites deployed, we need a unified evaluation dataset that covers diverse conditions across all sites.

## Reading Instructions

Before writing any code, read:
- .agents/handoff/P1-A01.md — CVAT setup, dataset organization
- .agents/handoff/P2-A01.md — cross-camera annotation
- .agents/handoff/P3-A02.md — Re-ID training data (once available)
- CONVENTIONS.md

## Implementation

1. Create scripts/annotation/aggregate_datasets.py:
   - Pull annotated data from multiple CVAT instances (one per site)
   - Normalize annotations to common format
   - Tag each sample with site_id, conditions (day/night/weather), camera_model
   - Merge into unified dataset
2. Create scripts/annotation/dataset_analysis.py:
   - Compute dataset statistics: samples per class per site, condition distribution, camera model coverage
   - Identify gaps: underrepresented conditions or classes
   - Generate coverage report
3. Create scripts/annotation/balanced_split.py:
   - Create balanced train/val/test splits ensuring each split has proportional representation from all sites and conditions
   - Version with DVC

## Constraints

- Must handle different CVAT versions across sites
- Annotations must be normalized to a single format
- Must pass `ruff check`

- scripts/annotation/dataset_analysis.py
- scripts/annotation/balanced_split.py
