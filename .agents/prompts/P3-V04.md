# P3-V04: Adaptive Transit Time

You are working in /repo/services/topology/. Implement TimescaleDB continuous aggregates for topology learning.

## Context

The MTMC service uses transit-time distributions to filter candidates. Currently these are static priors. This task learns them from observed cross-camera transitions.

## Reading Instructions

Before writing any code, read:
- .agents/handoff/P0-D05.md — topology graph, TransitionEdge with p50/p90/p99 fields
- .agents/handoff/P2-V02.md — MTMC service, how transit times are used in scoring
- services/topology/models.py — TopologyGraph, TransitionEdge
- services/db/models.py — topology_edges table
- CONVENTIONS.md

## Implementation

1. Create a TimescaleDB continuous aggregate on confirmed MTMC transitions:
   - Source: global_track_links joined with local_tracks for timing
   - Aggregate: per-edge (from_camera, to_camera), per-class: p50, p90, p99 transit time, sample_count
   - Refresh interval: every 1 hour
2. Create services/topology/adaptive_transit.py:
   - Read continuous aggregate results
   - Update topology_edges table with learned distributions
   - Blend with prior: weight = min(sample_count / 100, 1.0) — use prior until enough samples
3. Create Alembic migration for the continuous aggregate
4. Add API endpoint: GET /topology/{site_id}/transit-stats — return learned distributions

- Continuous aggregate must not impact write performance
