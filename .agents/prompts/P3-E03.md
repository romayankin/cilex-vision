# P3-E03: Drift Monitoring

You are working in /repo/scripts/monitoring/. Detect confidence distribution shift in production.

## Context

Over time, model performance may degrade due to changing conditions. This task detects drift by monitoring detection confidence distributions.

## Reading Instructions

Before writing any code, read:
- .agents/handoff/P1-V04.md — detection confidence output
- .agents/handoff/P1-O02.md — model-quality dashboard, confidence histograms
- docs/taxonomy.md — confidence thresholds
- CONVENTIONS.md

## Implementation

1. Create scripts/monitoring/drift_detector.py:
   - Query TimescaleDB for hourly confidence distribution per class per camera
   - Compare against baseline distribution (stored from initial deployment)
   - Use KL divergence or KS test to detect significant drift
   - Output drift score per class per camera
   - Trigger alert if drift exceeds threshold
2. Create scripts/monitoring/baseline_snapshot.py:
   - Capture current confidence distribution as baseline
   - Store in MinIO as JSON
3. Add Prometheus metrics: confidence_drift_score{class, camera_id}
4. Add Grafana panel to model-quality dashboard

## Constraints

- Must run as a periodic job (cron/timer)
- Do NOT modify inference worker
- Must pass `ruff check`

## Expected Deliverables

- scripts/monitoring/drift_detector.py
- scripts/monitoring/baseline_snapshot.py
