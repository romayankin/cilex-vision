# P2-O02: Storage Tiering

You are working in /repo/infra/. Implement MinIO lifecycle policies for hot/warm/cold storage tiers.

## Context

With increasing data from attributes, MTMC, events, and clips, storage costs need tiering. This implements the retention policies from the privacy framework (P0-X02).

## Reading Instructions

Before writing any code, read:
- .agents/handoff/P1-O01.md — MinIO deployment, existing bucket structure
- docs/privacy-framework.md — retention policies per data class
- .agents/handoff/P0-X01.md — cost model storage parameters
- CONVENTIONS.md

## Implementation

1. Define MinIO lifecycle policies in infra/minio/lifecycle-policies.json:
   - Hot tier (NVMe/SSD): raw frames, active embeddings — 7 days
   - Warm tier (HDD): event clips, thumbnails — 90 days
   - Cold tier (compressed archive): raw video archives — 30 days then delete
   - Debug traces: 30 days then delete
2. Create infra/minio/apply-lifecycle.py — script to apply lifecycle policies to all buckets
3. Create infra/minio/storage-report.py — script that queries MinIO admin API, reports per-bucket size, tier distribution, projected costs
4. Add Grafana dashboard panel to storage.json for tier utilization
5. Add Prometheus alerts for storage approaching capacity thresholds

## Constraints

- Do NOT modify any service code
- Lifecycle policies must be idempotent (re-runnable)
- Follow MinIO ILM (Information Lifecycle Management) API
