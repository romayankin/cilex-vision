## Implementation

Create a Python service that:
1. Consumes tracklets from Kafka `tracklets.local` topic
2. Applies quality gates: min 40px height, sharpness >50 Laplacian variance, brightness 30-220 mean, occlusion <40%
3. Detects IR/night mode by checking frame saturation (if mean_saturation <15, mark as IR and skip color classification)
4. Applies white-balance correction via cv2.xphoto.createSimpleWB()
5. Crops object from frame (fetch from MinIO)
6. Runs color classifier via Triton gRPC (vehicle_color, person_upper_color, person_lower_color)
7. Aggregates over track lifetime with confidence-weighted voting
8. Stores results in PostgreSQL track_attributes table
9. If quality never passes threshold, store 'unknown' — never guess

Files to create:
- main.py — Kafka consumer, orchestrates quality→classify→store pipeline
- quality_gate.py — all quality checks (height, sharpness, brightness, occlusion, IR detection)
- white_balance.py — white-balance normalization
- classifier_client.py — Triton gRPC client for color classifiers
- aggregator.py — confidence-weighted voting across track observations
- publisher.py — write results to PostgreSQL track_attributes
- config.py — Pydantic settings from YAML
- metrics.py — Prometheus metrics: attributes_classified_total{type,value}, quality_rejected_total{reason}, ir_frames_skipped_total
- Dockerfile
- tests/test_quality_gate.py — synthetic day/night/IR fixture images
- tests/test_aggregator.py — voting logic with multiple observations
