# P1-V04: Detection & Tracking Inference Worker

You are working in /repo/services/inference-worker/. Python service.

- main.py — Kafka consumer (frames.sampled.refs), detect→track→embed pipeline.
- detector_client.py — Triton gRPC client, send batch, receive detections, apply confidence threshold from taxonomy.md.
- tracker.py — Single-camera tracker (ByteTrack/BoT-SORT per bake-off). Per-camera state dict. Output: updated tracks with local_track_id, state, trajectory.
- embedder_client.py — Triton gRPC for Re-ID model on best frame per track. Store embedding, DO NOT activate MTMC.
- publisher.py — Detection protobuf to tracklets.local, Embedding to mtmc.active_embeddings (compacted), bulk detections to bulk-collector.
- debug_trace.py — sample 1-5% of inference paths. Store as JSON + frame ref in MinIO. Always trace low-confidence detections.

Metrics: detections_total{class}, tracks_active, tracks_closed, inference_latency_ms, embedding_latency_ms.
Tests: test_detector_client.py (mock Triton), test_tracker.py (synthetic sequences), test_debug_trace.py.
