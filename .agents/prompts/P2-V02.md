## Implementation

Create a stateful Python service sharded by site_id via Kafka consumer group (cooperative sticky assignor):
1. In-memory FAISS flat index for active-horizon embeddings
2. Scoring pipeline: (a) filter candidates by topology graph (adjacent cameras, within transit-time window), (b) embedding cosine similarity via FAISS, (c) color attribute consistency check, (d) class consistency, (e) transit-time likelihood from adaptive distribution
3. Assignment: greedy if <10 candidates, Hungarian if more
4. Checkpoint: every 60s snapshot FAISS index to local NVMe, every 5min to MinIO
5. On startup: restore from latest snapshot + replay mtmc.active_embeddings changelog
6. CRITICAL: include embedding_model_version in every record, never compare embeddings across version boundaries

Files to create:
- main.py — Kafka consumer, site_id sharding, startup restore
- matcher.py — scoring pipeline (topology filter → FAISS similarity → attribute check → assignment)
- faiss_index.py — FAISS index management (add, search, flush, rebuild)
- checkpoint.py — local NVMe + MinIO snapshot/restore
- topology_client.py — query topology graph for candidate filtering
- publisher.py — write global_tracks and global_track_links to PostgreSQL
- config.py — Pydantic settings
- metrics.py — match_rate, reject_rate, rebalance_duration_ms, checkpoint_size_bytes, faiss_index_size
- Dockerfile
- tests/test_matcher.py — synthetic embeddings with known matches
- tests/test_checkpoint.py — snapshot/restore cycle
