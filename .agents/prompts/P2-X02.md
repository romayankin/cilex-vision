# P2-X02: Operations Runbooks

You are working in /repo/docs/runbooks/. Create incident response, scaling, and backup procedures.

## Context

With the full Phase 2 pipeline deployed, operators need documented procedures for common operational scenarios.

## Reading Instructions

Before writing any code, read:
- .agents/handoff/P1-O01.md — infrastructure deployment, Ansible playbooks
- .agents/handoff/P1-O02.md — dashboards, alert rules
- .agents/handoff/P2-O01.md — MTMC infrastructure
- .agents/handoff/P2-O02.md — storage tiering
- docs/runbooks/model-rollout-sop.md — existing rollout runbook pattern
- CONVENTIONS.md

## Implementation

Create runbooks following the pattern in model-rollout-sop.md:

1. docs/runbooks/incident-response.md — for each alert type: what it means, how to diagnose, how to fix, escalation path
2. docs/runbooks/scaling.md — procedures for adding cameras, adding GPU nodes, scaling Kafka, scaling DB
3. docs/runbooks/backup-restore.md — what to back up (DB, MinIO, FAISS checkpoints, configs), how to restore, RPO/RTO targets
4. docs/runbooks/camera-onboarding.md — step-by-step adding a new camera: physical setup, ONVIF config, edge agent config, topology update, calibration
5. docs/runbooks/service-restart.md — per-service restart procedures, dependency order, state recovery

## Constraints

- Each runbook must be executable by an operator who is NOT an engineer
- Include specific commands, not just descriptions
- Reference dashboard URLs and alert names
