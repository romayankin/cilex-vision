# P4-V05: Multi-Site Portal

You are working in /repo/frontend/. Create a multi-tenant site management UI.

## Context

For commercial deployments with multiple customer sites, a portal is needed to manage and monitor all sites from a single interface.

## Reading Instructions

Before writing any code, read:
- .agents/handoff/P2-V05.md — existing frontend, Search UI
- .agents/handoff/P3-V03.md — admin UI (once available)
- services/db/models.py — sites table
- CONVENTIONS.md

## Implementation

1. /portal — site overview dashboard:
   - List all sites with health status (green/yellow/red)
   - Camera count, active alerts, storage usage per site
   - Click through to site-specific search/admin pages
2. /portal/sites/new — create new site wizard
3. /portal/sites/{site_id}/settings — site-specific configuration
4. /portal/comparison — cross-site metrics comparison
5. Multi-tenant auth: JWT includes site_ids[], user sees only authorized sites
6. Site context switcher in navigation bar

## Constraints

- Must support 50+ sites without performance degradation
- Reuse existing components from Search UI and Admin UI
- Must pass `next lint`
