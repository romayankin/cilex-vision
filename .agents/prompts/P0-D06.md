# P0-D06: Ingress Bridge Design Specification

You are working in /repo. Create docs/adr/ADR-001-ingress-bridge.md as an Architecture Decision Record.

Specify:
- CONTEXT: Edge agents publish to NATS JetStream. Central services consume from Kafka.
- DECISION: Dedicated Ingress Bridge service.
- RESPONSIBILITIES: (1) consume durably from NATS, (2) validate Protobuf schema via registry, (3) offload blobs >100KB to MinIO, (4) produce to Kafka with idempotent keys {site_id}:{camera_id}:{capture_ts}:{frame_seq}, (5) spool to NVMe on Kafka failure (max 50GB), (6) drain spool with rate limiting on recovery, (7) per-site rate limits (500 msg/s default), (8) live vs replay lane separation.
- DELIVERY GUARANTEE: At-least-once. Downstream deduplicates by key.
- FAILURE MODES: table (failure, detection, behavior, recovery).
- HEALTH/METRICS: /health, /ready, /metrics. List all Prometheus metrics.
- DEPLOYMENT: 2 instances min, 4-8 vCPU, 16-32GB RAM, 500GB-1TB NVMe spool.

Also create docs/diagrams/ingress-bridge-flow.mermaid (normal, Kafka outage, replay).
