# P0-D05: Camera Topology Graph Data Model

You are working in /repo/services/topology/. The topology_edges and cameras tables are in services/db/models.py (from P0-D04).

Create:
1. services/topology/models.py — Pydantic models: TopologyGraph, CameraNode, TransitionEdge with adaptive transit-time distributions (per edge, per class: p50_ms, p90_ms, p99_ms, sample_count, last_updated). Include zone_id on cameras.
2. services/topology/api.py — FastAPI router: GET /topology/{site_id}, PUT /topology/{site_id}/edges, POST /topology/{site_id}/cameras, DELETE /topology/{site_id}/cameras/{camera_id}.
3. services/topology/seed.py — seed a sample 4-camera site with plausible edges and broad transit-time windows (1s-120s).
4. docs/topology-schema.json — JSON Schema for import/export.
5. tests/test_topology.py — CRUD operations and graph queries (adjacent cameras, plausible downstream within time window).
