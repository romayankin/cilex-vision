# P2-V05: Search UI & Timeline

You are working in /repo/frontend/. Create a Next.js app with Tailwind CSS.

## Context

This task builds the customer-facing UI for searching events, viewing timelines, and following cross-camera journeys. It consumes the Query API (P1-V06).

## Reading Instructions

Before writing any code, read:
- .agents/handoff/P1-V06.md — query API endpoints, response schemas, auth model
- docs/taxonomy.md — object classes, attributes, event types for filter options
- docs/privacy-framework.md — RBAC roles, data access restrictions
- CONVENTIONS.md

## Implementation

Pages:
1. /search — faceted search: site, camera, time range, class, color, event type → paginated results with thumbnails
2. /timeline/{camera_id} — chronological detection timeline with video player synced to metadata overlay
3. /journey/{global_track_id} — cross-camera journey view: map/diagram of cameras visited with timestamps and clips
4. /admin — links to Grafana dashboards, camera management placeholder

Components:
- VideoPlayer (HLS.js)
- SearchFilters (faceted filter panel)
- ResultCard (detection/event result with thumbnail)
- Timeline (time-based visualization)
- JourneyMap (camera-to-camera path visualization)

API client auto-generated from /services/query-api/openapi.json using openapi-typescript-codegen. RBAC: check JWT role, hide redacted content for 'viewer' role. WebSocket/SSE for live detection updates on timeline page.

## Constraints

- Do NOT modify services/query-api/
- Use Next.js 14+ App Router
- Use Tailwind CSS for styling
- API client must be auto-generated, not hand-written
- Must pass `next lint`
