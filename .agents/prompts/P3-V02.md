# P3-V02: Shadow Deploy Tooling

You are working in /repo/infra/triton/ and /repo/scripts/shadow/. Implement Triton EXPLICIT mode shadow inference.

## Context

The model rollout SOP (P0-D09) specifies Stage 2 shadow deployment. This task builds the tooling to run new models in shadow mode alongside production.

## Reading Instructions

Before writing any code, read:
- docs/runbooks/model-rollout-sop.md — Stage 2 shadow deployment spec
- .agents/handoff/P0-D10.md — Triton config, EXPLICIT control mode
- .agents/handoff/P1-V04.md — inference worker, detector_client.py
- infra/triton/models/ — existing Triton model configs
- CONVENTIONS.md

## Implementation

1. Create infra/triton/shadow/ — shadow model repo directory structure
2. Create scripts/shadow/deploy_shadow_model.py:
   - Copy model to shadow Triton instance
   - Configure EXPLICIT control mode (model not loaded until explicitly requested)
   - Load model via Triton HTTP API
3. Create scripts/shadow/shadow_inference_worker.py:
   - Consume same frames as production inference worker
   - Run inference through shadow Triton instance
   - Publish results to shadow Kafka topics (tracklets.shadow, detections.shadow)
   - Do NOT affect production pipeline
4. Create scripts/shadow/compare_shadow.py:
   - Compare production vs shadow outputs: detection count agreement, confidence distribution delta, disagreement examples
   - Output comparison metrics to Prometheus

## Constraints

- Shadow inference must NEVER affect production pipeline
- Use Triton EXPLICIT control mode
- Shadow topics must be separate from production topics
- Must pass `ruff check`
