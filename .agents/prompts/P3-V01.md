# P3-V01: Model Retraining Pipeline

You are working in /repo/scripts/training/. Create a reproducible retraining pipeline.

## Context

With annotated data from CVAT (P1-A01, P2-A01) and evaluation baselines from bake-offs, we need an automated pipeline to retrain and evaluate models.

## Reading Instructions

Before writing any code, read:
- .agents/handoff/P1-E01.md — detector bake-off, MLflow experiment structure
- .agents/handoff/P1-A01.md — CVAT projects, dataset split script
- docs/bake-off-protocol.md — evaluation metrics and thresholds
- docs/runbooks/model-rollout-sop.md — four-stage rollout process
- CONVENTIONS.md

## Implementation

Create a reproducible pipeline:
1. pull_data.py — export latest approved annotations from CVAT, organize by class/condition
2. version_data.py — version dataset with DVC (dvc add data/training/v{N})
3. train.py — Hydra-configured PyTorch training script for detector fine-tuning, log to MLflow (params, metrics per epoch, artifacts)
4. evaluate.py — run on full eval dataset, compare with production baseline from MLflow, output per-class and per-condition deltas, flag regressions >2%
5. export.py — ONNX → TensorRT conversion
6. promote.py — register model in MLflow registry if approved

Makefile target: `make retrain`. One command, fully reproducible.

## Constraints

- Pipeline must be fully reproducible (DVC + MLflow)
- No regression >2% on any evaluation slice
- Must pass `ruff check`
