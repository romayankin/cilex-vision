# P2-V03: Event Engine

You are working in /repo/services/event-engine/. Create a Python service.

## Context

The Event Engine implements rule-based event detection from track state machines. It consumes tracklet updates from the inference worker (P1-V04) and produces structured events per the taxonomy (P0-D01).

## Implementation

1. Consume tracklets from `tracklets.local` Kafka topic
2. Maintain per-track state machines in memory:
   - ACTIVE → STOPPED: no movement for configurable duration (default 5s)
   - STOPPED → LOITERING: stopped for configurable duration (default 30s)
   - Track creation → entered_scene event
   - Track state EXITED/CLOSED → exited_scene event
   - Transition from STOPPED → ACTIVE → motion_started event
3. When an event triggers, emit an Event protobuf to `events.raw` Kafka topic
4. Write events to PostgreSQL events table
5. Include all required fields: event_id (UUID), event_type, start_time, end_time, duration_ms, track_ids, camera_ids, clip_uri (null initially — P2-V04 fills this), thumbnail_uri

Files to create:
- main.py — Kafka consumer, event loop
- state_machine.py — per-track FSM with configurable thresholds
- event_emitter.py — construct Event protobuf, publish to Kafka + write to DB
