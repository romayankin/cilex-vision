# P4-O02: Disaster Recovery

You are working in /repo/infra/. Implement backup, restore, and failover automation.

## Context

Production deployments require documented and tested disaster recovery procedures with defined RPO/RTO targets.

## Reading Instructions

Before writing any code, read:
- .agents/handoff/P1-O01.md — infrastructure components
- docs/runbooks/backup-restore.md — existing backup procedures (once available from P2-X02)
- docs/privacy-framework.md — data retention requirements
- CONVENTIONS.md

## Implementation

1. Create infra/backup/:
   - backup-db.sh — TimescaleDB/PostgreSQL backup (pg_dump + continuous archiving)
   - backup-minio.sh — MinIO bucket replication or mc mirror
   - backup-config.sh — configuration files, Ansible vars, Terraform state
   - backup-faiss.sh — FAISS checkpoint backup from MinIO
2. Create infra/restore/:
   - restore-db.sh — full restore from backup
   - restore-minio.sh — bucket restore
   - restore-full.sh — complete system restore from scratch
3. Create infra/failover/:
   - health-watchdog.py — monitor critical services, trigger failover
   - failover-kafka.py — Kafka broker failover procedures
   - failover-db.py — PostgreSQL failover to standby
4. Define RPO/RTO targets in docs/disaster-recovery.md:
   - RPO: <1 hour for metadata, <4 hours for video
   - RTO: <2 hours full restore

## Constraints

- Backup scripts must be idempotent and schedulable
- Restore must be tested (include test-restore.sh)
- Must pass `ruff check` on Python files
